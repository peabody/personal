Shell Accounts and Software

So you've got a shell account on someone's machine but you don't have root access and they don't have the software installed that you need? What can you do? Simple. Provided you have the diskspace, you can install software into your user folder. Here's how.

Step 1

Download and extract the tar ball somewhere. If the unix shell account you're using has your home folder mounted via nfs, it's nicer to actually extract it into /tmp.

Step 2

Configure the package so that it will install into your home folder. If the source package uses the GNU autoconf tools you should be able to do this from the configure script ./configure --prefix=/path/to/folder/to/install

Step 3

make && make install. If everything went smoothly, you should now be able to change your environment variables.

Step 4

Change your environment varibles. Set your PATH so that the bin folder of the folder you installed to is accessible. If your software installed a shared library, set LD_LIBRARY_PATH so that it includes the /lib/<softwarename> folder. This should do it, you can run the new software.
